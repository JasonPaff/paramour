**Breaking:** the scanner and artifact pipeline are now two-router (design-06 Group 3). `scanRoutes` is now the orchestrator — it takes `{ appDir?, pagesDir? }` and returns `{ appRoutes, pagesRoutes }` — with the App Router walker renamed to `scanAppRoutes` and a new `scanPagesRoutes` implementing the Pages Router rules (any file matching `pageExtensions` routes; `index.<ext>` maps to its directory; dynamic segments on files; top-level-only exclusion of `api/`, `_app`, `_document`, `_error`, `404`, `500`; `(group)`/`@slot`/`(.)x` are literal segments). `resolveAppDir` is replaced by the joint `resolveRouteDirs`: a root `app/` or `pages/` disables the `src/` variants for both routers, and an ignored `src/` dir containing page files is a hard config error. The emitted artifact now carries per-router registry members (`appRoutes` / `pagesRoutes`, either absent when its scan is empty); `withTypedRoutes` and the CLI (new `--pages-dir` flag; a route dir is required, but either one suffices) watch and generate over both dirs. Route collisions are errors instead of silent dedupes, exported as `RouteCollisionError`: two files mapping to one path (route-group twins, extension twins, `blog.tsx` + `blog/index.tsx`), a path present in both routers, different slug names at one position, and an optional catch-all beside its own base path. The CLI exits 2 and `withTypedRoutes` throws during config evaluation; a collision during watch logs loudly and keeps the last good artifact (drift reports now name the router per path). Regenerate committed artifacts with `paramour generate` — app-only projects produce byte-identical output.
